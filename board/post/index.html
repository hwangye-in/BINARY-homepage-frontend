Introduction to Binary Exploitation Techniques

Alice Dev Oct 24, 2023

Understanding the fundamentals of memory corruption is crucial for any security researcher. In this post, we explore the stack layout, how buffer overflows occur, and the basic mechanisms behind overriding the instruction pointer.

The Stack Layout

When a function is called, a stack frame is created to store local variables and function parameters. This structure is Last-In, First-Out (LIFO). The critical vulnerability often lies in how the program handles user input without proper bounds checking. If we can write past the allocated buffer, we can corrupt adjacent memory.

Controlling EIP

The Extended Instruction Pointer (EIP) determines which instruction the CPU executes next. By overflowing the return address on the stack, we can point EIP to our own injected code (shellcode).

// Vulnerable C code example

void vulnerable_function(char *str) {
char buffer[64]; strcpy(buffer, str); // No bounds check! }

Modern systems implement protections like ASLR (Address Space Layout Randomization) and DEP (Data Execution Prevention) to mitigate these attacks, but understanding the raw mechanic is the first step in bypassing them.
